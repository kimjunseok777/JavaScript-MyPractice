2. flex
        flexible layout
        : 유연하게 조작가능한 레이아웃

        flex는 '유연하다'라는 의미를 가지고 있으며, 빈 공간을 채우거나
        특정 요소를 수직 수평 정렬하고 어떠한 방향이나 위치든 유연하게 배치할 수 있는
        레이아웃을 의미한다

        ex) 모바일, 데스크탑 화면에 따라 요소의 위치가 바뀔 때 유연하게 대처하는 것 (반응형에 굉장히 좋다)

        반드시 "부모요소"에
        display: flex 를 선언 (자식 요소에 선언하는 것이 아니다)  -->  부모요소에 선언하면, 자식 요소에 영향이 가는 것이다

            justify-contents : 자식 요소에 "수평 정렬" 방식 
            (왼쪽 정렬, 가운데 정렬, 요소 사이 균등하게 벌어짐 등 여러가지 정렬 방식이 있다)  -->  ex) center , space between ... 등이 있다

            align-items :  자식 요소의 "수직 정렬" 방식
            (상단, 중단, 하단에 배치할 수 있음)

            flex-direction : 방향을 설정 - default 값은 수평이지만, 수직으로 세워서 flex속성을 사용  -->  ex) row, column 등이 있다  -->  자식 생성 방향을 수직으로 바꿀 수 있음 (기본값이 row 이다)

            flex-wrap : flex된 요소들의 "줄바꿈" 요소 (wrap, no-wrap) --> 원래는 수평으로 나열됨, 하지만 줄바꿈 가능 (기본값이 no-wrap 이다)

            gap : 각 요소들이 가지고 있는 여백의 크기를 조정할 수 있음
            ==> column-gap, grid-gap 등등 여러가지 gap이 있던데 잘 모르겠다 --> 질문!!

            (flex들 요소들은 "margin"을 사용하지 않고 "gap"을 사용한다 -> 부모한테 "gap"을 줘서 마진 줌 -> 요소들간의 간격을 정해줄 수 있다)

            gap 을 사용하는 이유는, flex 요소들간에만 여백을 줄 수 있기 때문이다  -->  그냥 margin 을 사용해버리면 요소 외에도 여백이 발생한다 (이것은 부모의 패딩값으로 정해주는 것이 깔끔하고 좋다)

---------------------------------------------------------------------------

            *자식 요소에 쓸 수 있는 속성 (위에 것들은 부모 요소에 줄 수 있는 속성)

                flex-basis : 정렬된 요소의 크기 (width) --> flex 의 자식들의 width 값을 더 유연하게 정해줄 수 있다

                flex-grow : 정렬된 요소가 '차지하는 비율' 입니다. 단, basis 속성이 있다면 해당
                값만큼 + 되기 때문에 grow 속성이 있는 경우 basis를 쓰면 안된다

                flex-shrink : 자식 요소가 부모 요소를 넘었을 때 줄어드는 비율
                                    (display flex는 자동으로 부모요소를 넘지 못하게 요소의 크기를 '균등하게' 줄입니다  -->  하지만 flex-shrink 또는 flex-wrap 을 사용해주면 요소의 크기를 유지할 수 있다)
                                    -만약에 shrink를 0 으로 설정하면 부모 요소의 크기를 넘을 수 있음 --> 자식 요소들이 균등하게 줄어들지 않고, 부모의 크기를 넘어서 튀어 나간다 (가로 스크롤 배너 만들 때 좋음)
                                    --> 사이트의 메인 상단부분에 좌우로 넘길 수 있는 배너 같은 곳에서 예시를 찾을 수 있다

                                    ==>  즉, 배너같은 역할을 해야한다면 flex-shrink : 0 을 주는 것이 좋다  ***
                                    ==>  반면에, 부모요소를 넘지 않고, 요소 크기가 유지되면서 줄바꿈이 되어야한다면 flex-wrap : wrap 을 주는 것이 좋다 (이 속성은 자식이 아닌 부모요소에 부여해준다)  ***
                                    ==>  flex-shrink : 0 으로 해주면 요소 크기 그대로 부모 크기를 넘을 수 있다

                ==>  flex - basis / grow / shrink 도 공부해보자  ***



                ) flex-grow : 1을 넣어주면 부모요소 크기만큼 다 찬다 (하나만 줄 수도 있고, 여러개 줄 수도 있다 --> 자식 요소중 하나만 넣어줘서 포인트를 주기에 좋다)
                                                                                                                                                                       --> 자식요소들이 부모요소의 너비를 비율로 맞추기에도 좋다 (반응형에 좋다)
                                        부모요소에 여백이 있을 때, flex-grow 를 1로 해준 자식요소가 부모의 여백을 일정하게 다 채워준다 (나머지 자식들과 대비 효과를 줄 수도 있다)

                ) flex-shrink : 0 을 넣어주면 요소 크기가 유지되면서 부모요소의 크기를 넘을 수 있다 (자식 요소들 다 넣어주는 것이 좋다 --> 가로로 스크롤 되는 배너 만들 때 좋다)
                                        0 을 넣는다는 것은 자식 요소들의 축소 비율을 0으로 만든다는 것이다

                ) flex-basis : flex 의 자식들의 width 값을 더 유연하게 정해줄 수 있다
                                       ex : 따로 width 값을 정해주지 않고, flex-basis : 100px 로 해준다고 가정 --> width 값이 100px 보다 작으면 100px 로 고정,
                                                                                                                                                                   더 크다면 100px 맞추지 않고 크기 그대로 유지

                ) flex-wrap : wrap 을 사용하면 자식 요소들이 크기를 유지하면서, 부모 요소의 크기가 넘을 것 같다면 줄바꿈을 한다 (부모요소에 주는 속성이다)
                                      ( 2~3 줄 짜리 행이 있는 list 를 만들 때 쓰면 좋다 )
